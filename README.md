# Gemini-for-Discord

Асинхронный Discord-бот, интегрированный с нейросетью Google Gemini (`gemini-3.1-flash-lite`), предназначенный для генерации текста, анализа истории чата, суммаризации обсуждений и код-ревью.

---

## 1. Что это такое?
Это Discord-бот, написанный на Python, который предоставляет пользователям доступ к возможностям языковых моделей Google Gemini непосредственно в текстовых каналах сервера. Бот оптимизирован для работы в условиях региональных ограничений за счет встроенной поддержки прокси-серверов и содержит отказоустойчивую архитектуру обработки ошибок API.

---

## 2. Какие фичи уже работают?

### Доступные команды бота:
* **`/ai <запрос>`** — Прямое обращение к модели Gemini. Поддерживает контекст обсуждения, если отправлена в качестве ответа (reply) на другое сообщение.
* **`/context_ai <запрос>`** — Анализ контекста. Бот сканирует последние 100 сообщений в текущем канале и формирует точечный ответ на основе найденной информации.
* **`/retell <N>`** — Саммаризация чата. Генерирует краткий пересказ последних `N` сообщений (по умолчанию 50), избавляя от необходимости читать всю историю вручную.
* **`/explain`** — Разбор сложной информации. Вызывается строго через reply на текстовое сообщение, код или формулу. Модель декомпозирует и объясняет логику целевого сообщения простым языком.
* **`/aihelp`** — Вывод интерактивной справки по командам проекта.

### Технические особенности:
* **Поддержка прокси:** Весь трафик к серверам Gemini и Discord API маршрутизируется через прокси для обхода блокировок.
* **Потокобезопасность:** Блокирующие вызовы SDK Gemini вынесены в отдельные потоки (`asyncio.to_thread`), что предотвращает зависание основного event loop бота.
* **Валидация вывода:** Системный промпт принудительно ограничивает размер ответа до 1950 символов, исключая ошибки падения Discord из-за превышения лимитов длины сообщения.

## 3. Что нужно для работы?

Для работы проекта необходимо настроить конфигурацию среды окружения. В корневой директории проекта должен быть создан файл `.env` (шаблон находится в `.env.example`) со следующими переменными:

```env
DISCORD_TOKEN=YOUR_DISCORD_TOKEN     # Токен вашего бота из Discord Developer Portal
GEMINI_TOKEN=YOUR_GEMINI_API_KEY     # Ключ API для доступа к Google Gemini
PROXY=YOUR_PROXY                     # URL-адрес прокси-сервера (например, http://ip:port)
---

## 4. Как это установить и запустить?

Инструкция для быстрого развертывания проекта с нуля:

### Шаг 1: Клонирование и подготовка директории
Убедитесь, что на сервере/компьютере установлен Python 3.10 или выше. Перейдите в корневую папку проекта.

### Шаг 2: Настройка виртуального окружения
Инициализируйте и активируйте изолированное виртуальное окружение:

# Создание окружения (папка .venv добавлена в .gitignore)
python -m venv .venv

# Активация для Linux / macOS:
source .venv/bin/activate

# Активация для Windows (Command Prompt):
.venv\Scripts\activate

### Шаг 3: Установка зависимостей
Установите внешние библиотеки, необходимые для работы клиентов Discord и Gemini:

pip install discord.py google-genai python-dotenv

### Шаг 4: Конфигурация окружения
Создайте файл .env на основе примера:

cp .env.example .env

Заполните созданный файл .env актуальными токенами и адресом прокси.

### Шаг 5: Запуск бота
Запустите основной исполняемый файл проекта main.py:

python main.py
